# 风格反向工程检查清单

三张清单，分别在**提取前 / 迁移前 / 输出前**执行。每张清单都是硬性门槛——任一条未满足就不要推进到下一步。

> 与 SKILL.md 步骤对应：
> - 提取前 → 在 Step 1 之前
> - 迁移前 → 在 Step 3 之后、Step 4 之前（**仅跨主体场景**）
> - 输出前 → 在 Step 4 完成、交付用户之前

---

## 一、提取前 Checklist（所有场景必做）

在开始 Step 1（L1-L4 分层观察）前，确认：

- [ ] 参考图清晰度足够（短边 > 512px），可辨识构图、光影、材质细节
- [ ] 参考图无明显水印 / 严重压缩损伤（会污染材质判断，尤其 grain / halation / noise 分析）
- [ ] 若用户提供多张图：
  - [ ] 已与用户确认这些图在**意图上属于同一风格集**（同一摄影师 / 同一电影 / 同一系列），而非混杂
  - [ ] 已切到 [`multi-image.md`](multi-image.md) 的多图工作流
- [ ] 已明确用户的**目标输出工具**：MJ / SD / Flux / 通用（决定 Step 4 走哪个 T5x 分支）
  - 用户未指定时 → 默认走 T5d 通用模板，再询问是否需要特化
- [ ] **若是跨主体场景**，已拿到新主体描述（主体是什么、要做什么）
- [ ] 已与用户对齐**默认产出**：关键词列表（视觉 DNA）。除非用户明确要 JSON 指纹或中文报告，不要主动给

---

## 二、迁移前 Checklist（仅跨主体场景必做）

在 Step 3 跨主体适配后、Step 4 出 prompt 前，确认：

- [ ] 已识别并标注 `transfer_risk_tags` 三类（universal / conditional / subject_bound）
- [ ] **L4 兼容性体检已过**：
  - 情绪基调、叙事视角、时代/流派锚定 与新主体不冲突
  - 如有冲突，已显式向用户报警并拿到拍板（换图 / 接受转向 / 选择性放弃）
  - 典型反例：悲凉挽歌式基调 × 欢快生日派对主题
- [ ] `subject_bound` 属性已逐条明确策略：**保留 / 降权 / 丢弃 / 替换**
- [ ] `conditional` 属性的"底层逻辑"被显式保留（不是简单丢弃）
  - 例：暖金调在深夜题材上 → 保留"低调 + 高光偏暖暗部偏冷"，换成霓虹色域
- [ ] **纵横比（aspect_ratio）** 在新主体上依然合理
  - 反例：纵向肖像构图强套 2.39:1 宽画幅 → 主体被严重压缩
- [ ] **Negative cues** 在新主体上依然有效
  - 反例：`no neon oversaturation` 用在赛博朋克新主体上要放宽
- [ ] 风险点汇总表已生成，每条 High 风险都有处理方案

---

## 三、输出前 Checklist（Step 4 产出后必做）

在把 prompt 交付用户前，自查：

### 通用项

- [ ] 关键词严格按**权重顺序公式**排列
  - `[主体] → [流派锚定] → [构图] → [光照] → [镜头] → [色彩] → [材质] → [氛围] → [后期] → [排除项] → [工具参数]`
- [ ] 无空词（`beautiful` / `amazing` / `masterpiece` / `best quality` / `stunning`）
- [ ] 所有描述都是**可验证**的具体特征（另一个人看着 prompt 能判断是否符合原图气质）
- [ ] Negative cues 已包含
- [ ] 主体是**新主体**，不是原图主体（跨主体场景特别容易残留 L1）

### Midjourney 专项（T5a）

- [ ] `--ar` 与 `aspect_ratio` JSON 字段一致
- [ ] 若有原图 URL，已用 `--sref <url>`
- [ ] `--sw` 取值符合预期（严格还原 300-700，主体适配 100-200，SREF 极简 default）
- [ ] `--stylize` 值符合题材（写实 50-150，艺术化 500-1000）
- [ ] 已输出三个变体（忠实复刻 / 主体适配 / SREF 极简）

### Stable Diffusion 专项（T5b）

- [ ] 权重词（> 1.0）总数 ≤ 8
- [ ] `(keyword:1.3)` 语法正确（不要写成 `keyword:1.3` 或 `(keyword)1.3`）
- [ ] Negative prompt 包含 `negative_cues` + 通用 neg
- [ ] 采样器、CFG、尺寸建议已给出
- [ ] 已输出两个变体（写实向 / 艺术向）

### Flux 专项（T5c）

- [ ] 是**自然语言长句**，不是标签堆
- [ ] 长度 60-120 词
- [ ] 信息密度前高后低
- [ ] 已给出 guidance_scale / steps 建议
- [ ] 已输出两个变体（摄影写实向 / 艺术绘画向）

### 通用模板专项（T5d）

- [ ] 给出至少 1 个完整 prompt 示例
- [ ] 跨主题场景额外给 2-3 个不同主体的样例 prompt（共享同一套 L3/L4，验证风格稳定性）

### 可选验证（仅用户要求时启用）

- [ ] 自检盲测（T6）：用户要求严谨验证时启用
- [ ] 中文配方报告（T7）：用户要求沉淀风格库时启用

---

## 清单使用铁律

1. **不要跳步**。每条清单都来自实战踩坑，跳过就是埋雷。
2. **硬性门槛**。未满足的条目要么补齐，要么显式向用户声明"此条未满足，风险 X"。
3. **L4 冲突优先报警，不要硬上**。这是迁移前 checklist 的最高优先级条目。
4. **主体适配优先于还原**。跨主体场景的核心目标不是"像原图"，而是"让新主体拥有原图的气质"。有取舍时，优先保 L3/L4，牺牲 L2。
