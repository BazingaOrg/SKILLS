# 风格反向工程检查清单

三张清单，分别在**提取前 / 迁移前 / 输出前**执行。每张清单都是硬性门槛——任一条未满足就不要推进到下一步。

---

## 一、提取前 Checklist（所有场景必做）

在开始 A1（L1-L4 分层观察）前，确认：

- [ ] 参考图清晰度足够（短边 > 512px），可辨识构图、光影、材质细节
- [ ] 参考图无明显水印 / 严重压缩损伤（会污染材质判断，尤其 grain / halation / noise 分析）
- [ ] 若用户提供多张图，已确认这些图在**意图上属于同一风格集**（同一摄影师 / 同一电影 / 同一系列），而非混杂
- [ ] 已与用户确认本次任务所属场景（S1 深度分析 / S2 跨主题迁移 / S3 多图共性 / S4 MJ SREF / S5 SD-Flux 精调）
- [ ] 已明确用户的**目标输出工具**（MJ / SD / Flux / 通用）——决定 A5 走哪个子模板
- [ ] 若是 S2/S3 场景，已拿到**新主体描述**（主体是什么、要做什么）

---

## 二、迁移前 Checklist（S2 / S3 场景必做）

在 A4 迁移适配度检查后、A5 提示词生成前，确认：

- [ ] 已识别并标注 `transfer_risk_tags` 三类（universal / conditional / subject_bound）
- [ ] `subject_bound` 属性已逐条明确策略：**保留 / 降权 / 丢弃 / 替换**
- [ ] 新主体与原图的 **L4 元艺术哲学兼容**
  - 典型冲突：悲凉挽歌式基调 × 欢快生日派对主题 → 此时应提醒用户"神韵与新主体本质冲突，建议换一张参考图或接受气质转向"
- [ ] **纵横比（aspect_ratio）** 在新主体上依然合理
  - 反例：纵向肖像构图强套 2.39:1 宽画幅 → 主体会被严重压缩
- [ ] **调色逻辑（palette.tonal_logic）** 在新主体上依然成立
  - 暖金调在深夜题材上需要替换（不是简单丢弃，而是保留"低调 + 高光偏暖 + 暗部偏冷"的底层逻辑，换成霓虹色域）
- [ ] **Negative cues** 在新主体上依然有效
  - 反例："no neon oversaturation" 用在赛博朋克新主体上要放宽
- [ ] 已计划执行 A6 自检盲测（不要跳过）

---

## 三、输出前 Checklist（A5 / A7 产出后必做）

在把 prompt 交付用户前，自查：

### 通用项
- [ ] 关键词严格按**权重顺序公式**排列
  - `[主体] → [流派锚定] → [构图] → [光照] → [镜头] → [色彩] → [材质] → [氛围] → [后期] → [排除项] → [工具参数]`
- [ ] 无空词（`beautiful` / `amazing` / `masterpiece` / `best quality` / `stunning`）
- [ ] 所有描述都是**可验证**的具体特征（可以让另一个人看着 prompt 判断是否符合原图）
- [ ] Negative cues 已包含
- [ ] 主体是**新主体**，不是原图主体（跨主题场景特别容易残留）

### Midjourney 专项
- [ ] `--ar` 与 `aspect_ratio` JSON 字段一致
- [ ] 若有原图 URL，已用 `--sref <url>`
- [ ] `--sw` 取值符合预期（严格还原 300-700，主体适配 100-200，SREF 极简 default）
- [ ] `--stylize` 值符合题材（写实 50-150，艺术化 500-1000）
- [ ] 已输出三个变体（忠实复刻 / 主体适配 / SREF 极简）

### Stable Diffusion 专项
- [ ] 权重词（> 1.0）总数 ≤ 8
- [ ] `(keyword:1.3)` 语法正确（不要写成 `keyword:1.3` 或 `(keyword)1.3`）
- [ ] Negative prompt 包含 `negative_cues` + 通用 neg
- [ ] 采样器、CFG、尺寸建议已给出
- [ ] 已输出两个变体（写实向 / 艺术向）

### Flux 专项
- [ ] 是**自然语言长句**，不是标签堆
- [ ] 长度 60-120 词
- [ ] 信息密度前高后低
- [ ] 已给出 guidance_scale / steps 建议
- [ ] 已输出两个变体（摄影写实向 / 艺术绘画向）

### 自检（强烈建议）
- [ ] 已执行 A6 盲测
- [ ] 若盲测发现 ⚠️ 或 ❌，已迭代修订（最多 2 轮）
- [ ] 若 2 轮后仍有偏差，已回到 T3 检查美学指纹源头

---

## 清单使用铁律

1. **不要跳步**。每条清单都来自实战踩坑，跳过就是埋雷。
2. **硬性门槛**。未满足的条目要么补齐，要么显式向用户声明"此条未满足，风险 X"。
3. **多图场景警惕**。S3 场景特别容易在「图片在意图上属于同一风格集」这一条翻车——用户可能把混杂的素材当作统一风格。
4. **主体适配优先于还原**。S2/S3 场景的核心目标不是"像原图"，而是"让新主体拥有原图的气质"。有取舍时，优先保 L3/L4，牺牲 L2。
