# 多图共性提炼专题

> 仅在用户提供 **≥ 2 张参考图** 且需要从中抽出共性风格时启用。单图场景请直接走 `SKILL.md` 的 Quick Path。

## 为什么不能简单"分别分析后拼接"

单图分别走完 T1（L1-L4 观察）后再把结果机械拼接，会出现两个问题：

1. **放大噪声**：每张图的 L2 表层耦合（具体配色、特定时辰）会以"叠加"形式进 prompt，画面质感扭曲。
2. **稀释共性**：真正稳定的 L3/L4 骨架被淹没在差异里，"神韵"反而消失。

正确做法是：**先单图各自跑 T1，然后跑 T2 求交集，把异集排除或降权**。

## 多图工作流

```
1. 对每张图独立执行 T1（L1-L4 分层观察）
   └─ 这一步与单图无差别，输出 N 份独立笔记

2. 执行 T2（共性提取）
   └─ 输出三类特征：共性 / 高频 / 异常

3. 用"共性 + 高频"作为输入，跑 T3 输出美学指纹 JSON
   └─ 异常特征不进 JSON，可作为变体备注

4. 之后的 Step 3（适配）/ Step 4（出 prompt）与单图流程一致
```

## T2 三类特征的处理规则

| 特征类型 | 定义 | 处理 |
|---|---|---|
| **共性特征** | 所有 N 张图都明确具备 | 风格库的"骨架"，**必进**最终 prompt |
| **高频特征** | 出现在 ≥ 60% 图中 | 次级特征，**可选**进 prompt（标注出现率）|
| **异常特征** | 仅单张图具备 | 二选一处理：(a) **排除**——偶然非风格性；(b) **作为变体标签保留**——风格允许的自由度 |

## 输入质量硬门槛

启动多图流程前，**必须**先确认：

- [ ] 这些图在**意图上属于同一风格集**（同一摄影师 / 同一系列 / 同一电影 / 同一作品集）
- [ ] **不是用户从不同来源凑出来的混杂素材**
- [ ] N ≥ 2 且 ≤ 8（更多图边际收益递减，反而稀释判断）

> 如果用户的多图明显不属于同一风格集，**显式向用户报警**——硬上的结果是抽出"最大公约数"，可能只剩"都是数码摄影"这种无价值结论。建议用户拆成多个独立任务，或挑出最能代表他想要风格的 1-3 张作为输入。

## 与 SKILL.md 主流程的差异速查

| 步骤 | 单图 | 多图 |
|---|---|---|
| Step 1 观察 | T1 跑 1 次 | T1 跑 N 次 |
| Step 1.5 共性提取 | — | **新增 T2** |
| Step 2 美学指纹 | T3 直接输出 | T3 仅基于共性+高频，异常特征单独标注 |
| Step 3 适配 | 与单图一致 | 与单图一致 |
| Step 4 出 prompt | 与单图一致 | 与单图一致 |

**铁律**：异常特征不进 T3 JSON。需要保留时，单独以 `variant_tags` 字段挂在 JSON 旁边，让用户决定是否启用。
